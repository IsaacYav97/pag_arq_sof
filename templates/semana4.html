Semana 4: Arquitecturas Distribuidas y Observabilidad

El salto del monolito a la red. Cuando los sistemas crecen exponencialmente, la comunicación y la visibilidad se vuelven el problema principal. Pasamos de tener un "edificio gigante" a una "ciudad de edificios interconectados".

🌐 Conceptos Clave

🧩 Microservicios

Descomposición Funcional: Cada servicio tiene un solo dominio (Bounded Context) para reducir el radio de explosión ante fallos.
Despliegue Independiente: Permite subir a producción el servicio de "Pagos" sin esperar al equipo de "Catálogo".

👁️ Observabilidad

No es solo monitoreo. Es entender el estado interno de un sistema distribuido basándose en sus salidas externas.
Ecosistema: Prometheus (Métricas), ELK (Logs), Jaeger (Trazas Distribuidas).

📜 Datos Distribuidos

Database-per-service: Nunca compartir bases de datos entre servicios para evitar acoplamiento oculto.
CQRS: Separar las consultas (lectura) de los comandos (escritura).

⚖️ Gestión de Datos: 2PC vs SAGA

Característica 2-Phase Commit (2PC) Patrón SAGA
Consistencia Fuerte (Atómica) Eventual
Rendimiento / Escalabilidad Baja (Bloquea recursos) Alta (Operaciones Asíncronas)
Manejo de Fallos Rollback automático de DB Transacción Compensatoria

⚙️ Laboratorio Teórico: Transacciones Compensatorias

En un sistema distribuido (E-commerce), si el último paso falla, no puedes hacer un "Rollback" mágico en SQL porque los pasos anteriores ocurrieron en bases de datos distintas. SAGA resuelve esto.

Ver Flujo de Fallo SAGA (Compra de Laptop)
  • ✅ Paso 1 (Servicio de Inventario): Reserva 1 Laptop HP en la base de datos de inventario.
  • ✅ Paso 2 (Servicio de Pagos): Cobra $800 a la tarjeta a través del procesador de pagos.
  • ❌ Paso 3 (Servicio de Envíos): Falla porque el código postal ingresado no tiene cobertura.
  • 🔄 SAGA ACTIVA LA COMPENSACIÓN (Hacia atrás):
    • Compensar Paso 2: El sistema llama a la API del banco para emitir un reembolso de $800 al usuario.
    • Compensar Paso 1: El sistema devuelve la Laptop HP al inventario disponible.
