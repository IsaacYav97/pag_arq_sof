Semana 3: Estilos Estructurales y Clásicos

Organizando la complejidad mediante niveles de abstracción. Los estilos estructurales definen cómo se dividen los componentes principales de un sistema y cómo se comunican entre sí.

🏗️ Los 3 Estilos Clásicos

🥞 Arquitectura en Capas (N-Tier)

El estándar de la industria. Organiza el sistema en responsabilidades horizontales (Presentación, Negocio, Datos).

  • Ventaja: Alta mantenibilidad y seguridad por aislamiento.
  • Trade-off: Riesgo de "Sinkhole Pattern" (peticiones que atraviesan capas sin hacer nada, afectando el rendimiento).

🧩 Microkernel (Plug-ins)

Sistemas evolutivos. Un núcleo central (Core) mínimo, rodeado de módulos independientes que añaden funciones.

  • Ventaja: Flexibilidad extrema (ej. VS Code, Navegadores web).
  • Trade-off: Complejidad al gestionar versiones y dependencias de los plug-ins.

🧠 Pizarra (Blackboard)

Inteligencia colectiva. Varios "Sistemas Expertos" miran una pizarra común y actúan cuando ven datos que saben resolver.

  • Ventaja: Ideal para IA y problemas no deterministas.
  • Trade-off: Comportamiento difícil de predecir y testear.

📊 Comparativa de Estilos

Estilo Pilar de Calidad Caso de Uso Ideal Dificultad
Capas Mantenibilidad Apps Empresariales / CRUDs Baja
Microkernel Flexibilidad IDEs, Navegadores, Editores Media
Pizarra Fiabilidad (IA) Sistemas de Diagnóstico / IA Alta

💻 Laboratorio: Patrón Microkernel en Python

Observa cómo el CoreSystem no sabe nada sobre procesar pagos, simplemente expone una forma de registrar Plugins. Esto permite agregar "Pago con Cripto" mañana sin modificar el núcleo.

microkernel.py 

# 1. CORE SYSTEM (El Núcleo)
class CoreECommerce:
    def __init__(self):
        self.plugins_pago = {}

    def registrar_plugin(self, nombre, plugin):
        self.plugins_pago[nombre] = plugin
        print(f"[*] Plugin '{nombre}' registrado exitosamente.")

    def procesar_checkout(self, metodo, monto):
        if metodo in self.plugins_pago:
            self.plugins_pago[metodo].pagar(monto)
        else:
            print(f"[!] Error: Método de pago '{metodo}' no soportado.")

# 2. PLUG-INS (Funcionalidades extensibles)
class PluginPayPal:
    def pagar(self, monto):
        print(f"$$$ Procesando ${monto} a través de la API de PayPal.")

class PluginStripe:
    def pagar(self, monto):
        print(f"$$$ Procesando ${monto} con tarjeta de crédito vía Stripe.")

# 3. EJECUCIÓN (Conectando las piezas)
app = CoreECommerce()
app.registrar_plugin("paypal", PluginPayPal())
app.registrar_plugin("stripe", PluginStripe())

print("\n--- Iniciando Compras ---")
app.procesar_checkout("paypal", 150.00)
app.procesar_checkout("cripto", 500.00) # Este fallará porque no hay plugin
Terminal Output:
[*] Plugin 'paypal' registrado exitosamente.
[*] Plugin 'stripe' registrado exitosamente.

--- Iniciando Compras ---
$$$ Procesando $150.0 a través de la API de PayPal.
[!] Error: Método de pago 'cripto' no soportado.
